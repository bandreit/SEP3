SEP3 API Documentation

User

The User is sent back and forth as json.
The json format is seen in the example below:
{
    "Username": "Bob", 
    "Password": "123456"
}
API Request Sub-path Description Exceptions
/validate GET
This request will return a User.
This request takes an optional number of the following request parameters:
  • username
  • password
Example:
  • .../validate
  • .../validate?username=Mike
  • .../validate?username=Anne&password=aloha
 May return 200 if the User is found.
May return 404 Not Found, if the provided cridentials do not match any users.
May return 401 Unauthorized, if you cannot access the page.
May return 500 Internal Server Error, if something fails.
